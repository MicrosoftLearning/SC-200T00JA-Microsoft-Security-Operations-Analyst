# モジュール 3 - ラボ 1 - 演習 1 - Azure Defender の有効化

## ラボ シナリオ

あなたは、Azure Defender を使用してクラウド ワークロード保護を実装している企業に勤務するセキュリティ オペレーションアナリストです。  このラボでは、Azure Defender を次のように構成します。

### タスク 1: Azure ポータル にアクセスし、サブスクリプションを設定します。

このタスクでは、このラボと今後のラボを完了するために必要なAzureサブスクリプションを設定します。 

1. 管理者として WIN1 仮想マシンにログインします。パスワードは **Pa55w.rd** です。  

2.  以前のラボでインストールしていない場合は、ブラウザーを開き、新しい Microsoft Edge ブラウザーを検索してダウンロードし、インストールします。新しい Edge ブラウザーを起動します。

3.  Edge ブラウザーで Azure ポータルに移動します https://portal.azure.com

4. **サインイン** ダイアログボックスで、ラボ ホスティング プロバイダーの提供した管理者ユーザー名のテナント電子メール アカウントをコピーして貼り付け、**「次へ」** を選択します。

5. **パスワードの入力** ダイアログ ボックスで、ラボ ホスティング プロバイダーの提供した管理者のテナント パスワードをコピーして貼り付け、**サインイン** します。

6. Azure ポータルの検索バーに 「**サブスクリプション**」と入力して「サブスクリプション」をクリックし、サブスクリプションページに移動します.

7. 「**追加**」を選択します。

8. プランの一覧で「その他すべてのオファー」を展開し、「**Azure Pass スポンサープラン**」の「資格の確認」をクリックします。

9. 「標準オファーを選択します」をクリックして、サブスクリプションを有効化します。

**注:** **標準オファー**が無効の場合は、トレーナーに相談してください

**注:** サブスクリプションを使用可能になるまで、最大 30 分かかる場合があります。 

### タスク 2: Log Analytics ワークスペースを作成する。

このタスクでは、Azure Defender で使用する Log Analytics ワークスペースを作成します。

1. Azure ポータルの検索バーに 「*Log Analytics*」 を入力し 「**Log Analytics ワークスペース**」を選択します。.

2. コマンド バーから 「**+ 作成**」を選択します。

3. リソース グループの「**新規作成**」 を選択します

4. *rg-AzureDefender* と入力します。

5. 名前には、 *uniquename-AzureDefender* のように一意の名前を入力します。

6. 「**レビュー + 作成**」を選択します。

7. ワークスペースの検証が完了したら、「**作成**」を選択します。

**注:** 新しいワークスペースがプロビジョニングされるのを待ちます。これには数分かかる場合があります。

### タスク 3: Azure Defender を有効にする。

このタスクでは、Azure Defenderを有効にして構成します。

1. Azure ポータルの検索バーに「*セキュリティ*」を入力し**セキュリティセンター**を選択します。

2. セキュリティセンターの**はじめに**ページで、 **アップグレード**セクションに移動し、サブスクリプションが選択されていることを確認してから、画面の一番下にある「**アップグレード**」ボタンを選択します。

3. 次のページでは何もしません。

4. ポータルメニューの**管理**領域から「**価格と設定**」を選択します。 

5. サブスクリプションを選択します。 

6. リソースと料金を確認します。  **サーバー** を **オフ**にして、「**保存**」を選択します。

**注:** どのリソースが自動的にカバーされるか、および関連する料金を理解しましょう。ここでは、あえて**Azure Defender for Servers**を無効にしています。この目的は、Azure サブスクリプションのコストを管理するためです。通常は、有効のままにしておくことをお勧めします。

7. **設定**領域から「**自動プロビジョニング**」を選択します。

8. 自動プロビジョニング-拡張機能を確認します。  **Azure VM の Log Analytics エージェント** が**オフ**になっていることを確認します。 

9. セキュリティセンターポータルに戻り、「**価格と設定**」を再度選択します。

10. 前に作成したワークスペース ID を選択します。**uniquename-AzureDefender**

11. サーバープランがまだオフになっていない場合は、サーバーを**オフ**にして、「**保存**」を選択します。


### タスク 4: オンプレミスのサーバーに Azure Arc をインストールする。

オンプレミスサーバーのオンボーディングを簡単にするため。  Azure Arc をインストールすると、Azure がオンプレミスサーバーを管理できるようになります。

このタスクでは、オンプレミスサーバーに Azure Arc をインストールします。

1. 管理者として WINServer 仮想マシンにログインします。パスワードは **Passw0rd!**。 

2. Edge ブラウザーで Azure ポータルに移動します https://portal.azure.com

3. **サインイン**ダイアログボックスで、ラボ ホスティング プロバイダーから提供され**たテナントの電子メール**アカウントをコピーして貼り付け、「**次へ**」を選択します。

4. **パスワードの入力**ダイアログ ボックスで、ラボ ホスティング プロバイダーから提供された**テナントパスワード**をコピーして貼り付け、**「サインイン」** を選択します。

5. Azure ポータルの検索バーに 「*Azure Arc*」 と入力し 「**Azure Arc**」 を選択します。.

6. Azure Arc ページメニューから「 **サーバー**」を選択します。

7. **+ 追加** を選択します。

8. 単一サーバーの追加セクションで **スクリプトの生成** を選択します。

9. **次へ:リソースの詳細 >** を選択します。

10. 先ほど作成したリソース グループを選択します。ヒント: *rg-AzureDefender*

**注** リソースグループをまだ作成していない場合。別のタブを開き、リソースグループを作成します。

11. **次へ:タグ >** を選択します。

12. サブスクリプションを登録で、「**登録**」をクリックします。
 
13. **次へ:** を選択します。**スクリプト** をクリップボードにコピーします。

**注** 処理が完了するまで3分待ちます

14. 「**ダウンロード**」 ボタンの横にある**クリップボード**ボタンをクリックして、スクリプトをクリップボードにコピーする。

15.PowerShell ISE を管理者として実行します。  

16.PowerShell ISE のエディタにスクリプトを貼り付けます。

19. プロンプトで *Set-ExecutionPolicy -ExecutionPolicy Unrestricted* を入力し Enter キーを押します。

20. 「すべて続行」をクリックして続行します。

21. スクリプトを実行します。

23. 画面の指示に従って、デバイスの登録を完了します。これには、デバイスの認証が含まれます。

24. Azure Arc ポータル ページで、サーバーが追加されたことを確認します。

25. サーバー名が表示されるまで「**更新**」を選択します。

**注** これには数分かかる場合があります。

### タスク 5: オンプレミスサーバーを保護します。

このタスクでは、必要なエージェントを Windows Server に手動でインストールします。

1. WinServer で、Azure Security Center に移動し、**はじめに**ページを選択します。

2. 「**はじめに**」タブを選択します。

3. 「非 Azure サーバーの追加」セクションで「**構成**」を選択します。

4. 前に作成したワークスペースの横にある「**アップグレード**」を選択します。  ヒント: unique-AzureDefender 

5. 前に作成したワークスペースの横にある「**＋サーバー追加**」を選択します。

6. 「**Download Windows Agent (64 bit)**」 を選択します。

7. ダウンロードしたファイルを実行します。

8. エージェント設定オプションのウィザードページが表示されるまで「**次へ**」を選択し、「**エージェントをログ分析（OMS）に接続する**」を選択して、「**次へ**」を選択します。

9. 必要に応じて、Azure ポータルからワークスペース ID と主キーをコピーしてウィザードページのフィールドに貼り付け、「**次へ**」を選択します。

10. インストールを続行します。そして、完了したら「**完了**」を選択します。

11. Security Center ポータル ページに移動し、「**インベントリ**」を選択します。

12. サーバーが一覧に表示されます。表示されない場合はブラウザを更新してください。

**注** サーバーは保護されていないと表示されるはずです。  サーバーの Azure Defender プランをオフにしたので、これは正しいです。
